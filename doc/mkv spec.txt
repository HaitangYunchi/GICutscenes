///
Note on Unknown fields from EBML

There was a problem when field lengths had particular values, which were in fact equal to 2^(n*7)-1
The EBML standard considers that those values define unknown values/fields/data, and this can cause an entire cluster to be corrupted.
A workaround found is to increment the length by one before transforming it into a varint, then decrease the value by one. Therefore, the length number has an additional byte, and does not belong to the particular values.

Sources : https://github.com/ietf-wg-cellar/ebml-specification/blob/master/specification.markdown#unknown-data-size
